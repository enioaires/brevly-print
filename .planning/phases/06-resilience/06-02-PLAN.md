<objective>
Lay the crash-recovery + retry-enqueue foundation (RES-04, and the enqueue half of RES-01).
This is the vertical slice that makes a failed print RECOVERABLE: after this plan, a print
failure no longer silently leaves a job at 'pending' — it saves the fetched ESC/POS bytes to
`retry_queue` and marks the row 'printing' so both the poll loop (Wave 2 retry task) and boot
crash recovery can find it.

Purpose: Without migration v2 no code can write status='printing' (v1 CHECK rejects it), and
without the print_worker changes there is nothing in retry_queue to retry. This plan is the
prerequisite for the retry task (06-03).
Output: `config_store.rs` migration v2; `print_worker.rs` fence + retry_queue insert. Turns the
Wave-0 config_store_test user_version/printing assertions GREEN.

This plan does NOT touch health state, does NOT add the retry task, and does NOT touch the
Pusher/pending-pull path. Those are 06-03 and 06-04.
</objective>

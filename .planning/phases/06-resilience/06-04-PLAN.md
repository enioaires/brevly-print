<objective>
Implement offline recovery (RES-03). When the agent reconnects to Pusher after an internet
outage, Noren has queued the missed jobs server-side. This plan pulls those jobs via
`GET /api/agent/jobs/pending` immediately after every successful re-subscription, dedups them
against the existing `printed_jobs` fence, and forwards genuinely-new jobs to the print worker —
so no ticket is lost during an outage.

Purpose: RES-03 is the internet-outage half of Phase 6 resilience (the retry task handles the
printer-outage half). It is an isolated HTTP + WebSocket-loop change with no overlap with the
retry-path files, so it runs in parallel with 06-02.
Output: `noren_client.rs` gains `fetch_pending_jobs()` + `PendingJob` + `pub(crate)
validate_job_id`; `pusher/client.rs` gains the pending-pull call. Turns the Wave-0
pending_jobs_test stubs GREEN (after removing their #[ignore]).

CRITICAL CONTRACT CORRECTION: The Noren response is a WRAPPER OBJECT, not a bare array. See
interfaces below — 06-RESEARCH.md's assumed `Vec<PendingJob>` shape is WRONG; the real endpoint
returns `{ "jobs": [ {...} ] }`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Noren → fetch_pending_jobs response | Untrusted network input; job_id values flow toward SQLite + print worker |
| pending job_id → insert_print_job / print worker | Path-traversal / injection surface if job_id is not validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-07 | Tampering | job_id from GET /api/agent/jobs/pending (path traversal — CR-02) | mitigate | validate_job_id() called on every job.job_id in the pusher pending-pull block BEFORE insert_print_job / any forwarding; rejects `/ . \ ? # % \0` and empty |
| T-06-08 | Information Disclosure | agent_token in fetch_pending_jobs logs (T-02-02) | mitigate | token passed only via .bearer_auth(); no eprintln!/format! in fetch_pending_jobs or the pusher call site includes the token |
| T-06-09 | Denial of Service | pending pull blocking the ping/pong timer | mitigate | pull runs BEFORE the inner select! loop (not inside it); forwarding uses non-blocking try_send (WR-04) so the ping timer is never starved |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages this plan (all deps present since prior phases per RESEARCH Package Legitimacy Audit) |

# Phase 2 — UI Design Contract: Activation Window

> Visual and interaction contract for the Brevly Print activation window.
> Rendered with raw egui 0.35 (egui-winit + egui-wgpu) on winit 0.30.
> NO HTML/CSS/Tailwind. All measurements in egui logical points. All colors as Color32.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Palette

All colors expressed as `Color32::from_rgb(r, g, b)`. Apply via `egui::Visuals` overrides
in the activation window constructor, before `ctx.set_visuals(visuals)`.

| Role | Hex | Color32 | 60/30/10 | Usage |
|------|-----|---------|----------|-------|
| Background (dominant) | `#1A1A1A` | `Color32::from_rgb(26, 26, 26)` | 60 % | Window background (matches Phase 1 spike wgpu clear color `r:0.1 g:0.1 b:0.1`) |
| Panel / surface (secondary) | `#262626` | `Color32::from_rgb(38, 38, 38)` | 30 % | Widget fill, `fill` on `Frame::group` wrapping the form |
| Primary text | `#F5F5F5` | `Color32::from_rgb(245, 245, 245)` | — | `visuals.override_text_color` for body text and labels |
| Secondary / muted text | `#A3A3A3` | `Color32::from_rgb(163, 163, 163)` | — | Hints, placeholder, caption, re-activation banner |
| Accent (action) | `#2563EB` | `Color32::from_rgb(37, 99, 235)` | 10 % | "Ativar", "Imprimir teste", "Salvar ativação" — non-destructive primary buttons only |
| Accent hover | `#1D4ED8` | `Color32::from_rgb(29, 78, 216)` | — | `hovered().bg_fill` on accent buttons |
| Accent pressed | `#1E40AF` | `Color32::from_rgb(30, 64, 175)` | — | `active().bg_fill` on accent buttons |
| Accent disabled | `#1E3A6E` | `Color32::from_rgb(30, 58, 110)` | — | `bg_fill` when button disabled; pair with muted text `#616161` |
| Error / destructive | `#EF4444` | `Color32::from_rgb(239, 68, 68)` | — | Inline error text (D-04, D-12) AND the destructive re-bind confirm button fill (D-02) |
| Success / confirmation | `#22C55E` | `Color32::from_rgb(34, 197, 94)` | — | Test-print "funcionou?" confirmation line (D-08) |

Accent reserved for: the three non-destructive primary action buttons ("Ativar",
"Imprimir teste", "Salvar ativação"). No other interactive element uses the accent fill.
The one destructive action in this phase — "Confirmar migração" (D-02, invalidates another
machine's token) — uses the Error/destructive red fill `#EF4444`, NOT accent blue, so it is
visually distinct from safe primary actions.

### Applying Visuals (implementation note for executor)

```rust
let mut visuals = egui::Visuals::dark();
visuals.panel_fill          = Color32::from_rgb(26, 26, 26);   // background
visuals.window_fill         = Color32::from_rgb(26, 26, 26);
visuals.faint_bg_color      = Color32::from_rgb(38, 38, 38);   // widget fill
visuals.override_text_color = Some(Color32::from_rgb(245, 245, 245));
// Accent button states are applied per-widget via ui.visuals_mut() or custom style
// rather than globally, so non-accent buttons keep the default dark styling.
ctx.set_visuals(visuals);
```

## Spacing Scale (egui logical points)

egui uses logical points; all values are multiples of 4.

| Token | Points | egui API | Usage |
|-------|--------|----------|-------|
| xs | 4 | `ui.add_space(4.0)` | Icon gaps, hint text gap below field |
| sm | 8 | `ui.add_space(8.0)` | Gap between label and widget |
| md | 16 | `ui.add_space(16.0)` | Default spacing between form groups |
| lg | 24 | `ui.add_space(24.0)` | Spacing between major sections (header → form, form → buttons) |
| xl | 32 | `ui.add_space(32.0)` | Top/bottom padding inside CentralPanel |
| 2xl | 48 | — | Not used in this single-screen layout |

Form inner padding: `Frame::group(ui.style()).inner_margin(Margin::same(16.0))`.
Button row bottom margin from window edge: 24 pt.
Horizontal content margin (left/right inset from window edge): 32 pt.
Frame inner margins (banner, empty-state box): 12 pt (multiple of 4).

Exceptions:
- Touch/click target minimum for all buttons: 36 pt height (= 4 × 9, a multiple of 4).
  egui default Button is ~28 pt; add `ui.style_mut().spacing.interact_size.y = 36.0` at
  window scope. Justified: non-technical restaurant owner needs a comfortable click target.
- `"Atualizar lista"` secondary button: same 36 pt height as primary buttons.

## Typography

All sizes in egui logical points. egui maps these via `TextStyle` → `FontId`.
Set via `ctx.style_mut(|style| { style.text_styles = ... })`.

**Exactly 2 font weights** are used across the entire window: Regular (400) and Bold (700).
No SemiBold or other intermediate weight is declared.

| egui TextStyle | Size (pt) | Weight | Line Height approx. | Usage |
|----------------|-----------|--------|----------------------|-------|
| `Heading` | 20 | Bold (700) | 1.2 × size = 24 pt | "Brevly Print" header text, section labels |
| `Body` | 15 | Regular (400) | 1.5 × size = 22 pt | Field labels, ComboBox items, general text |
| `Button` | 15 | Bold (700) | 1.2 × size = 18 pt | Button labels ("Ativar", "Salvar ativação", etc.) |
| `Small` | 13 | Regular (400) | 1.4 × size = 18 pt | Inline error text, hint text, muted caption, re-activation banner |

Weight rationale: buttons use Bold (700) — the same weight as Heading — to signal
affordance without introducing a third weight. Bold on 15 pt labels over a dark background
provides strong contrast for a non-technical owner. Distinct sizes (20 / 15 / 13) carry the
hierarchy; weight is only Regular-vs-Bold.

Contrast: all text on `#1A1A1A` background must meet WCAG AA 4.5 : 1.
- Primary text `#F5F5F5` on `#1A1A1A` → contrast ≈ 17 : 1 (AAA).
- Secondary/muted `#A3A3A3` on `#1A1A1A` → contrast ≈ 5.7 : 1 (AA pass).
- Error red `#EF4444` on `#1A1A1A` → contrast ≈ 4.7 : 1 (AA pass).
- White `#FFFFFF` label on accent button `#2563EB` → contrast ≈ 6.7 : 1 (AA pass).
- White `#FFFFFF` label on destructive red `#EF4444` → contrast ≈ 3.4 : 1; use Bold 15 pt
  (large-text AA threshold 3 : 1 — pass).

### Serial Number Field (D-03)

D-03 defers style (segmented vs freeform) to the planner pending the real Noren serial format.
The visual contract covers both cases:

**Freeform (variable-length serial):**
```rust
let response = ui.add(
    egui::TextEdit::singleline(&mut state.serial_input)
        .hint_text("Cole ou digite o serial")
        .desired_width(f32::INFINITY)
);
```

**Segmented / masked (fixed-length serial, e.g. XXXX-XXXX-XXXX):**
- Render as a single `TextEdit::singleline` with a hint showing the pattern (e.g.,
  `"XXXX-XXXX-XXXX"`), applying a character filter to uppercase + dashes only.
- The planner selects this path once the serial format is confirmed.

Common contract for both:
- Label above: `"Serial"` in `Body` weight, primary text color.
- `hint_text` rendered in muted text `#A3A3A3`.
- On `response.lost_focus() && ui.input(|i| i.key_pressed(egui::Key::Tab))` → move focus
  to ComboBox.
- On `Enter` key press while field focused → trigger "Ativar" action (same as clicking
  the button).
- Paste (Ctrl+V) must work — egui handles this natively; no special code needed.

**Inline error below serial field** (D-04, D-12):
```rust
if let Some(err) = &state.serial_error {
    ui.add_space(4.0);
    ui.label(
        RichText::new(err)
            .text_style(TextStyle::Small)
            .color(Color32::from_rgb(239, 68, 68))
    );
}
```
Error text replaces the hint gap — it always occupies `Small` line height (reserve 18 pt
minimum) so the layout does not shift when the error appears. Use
`ui.allocate_space(vec2(available_width, 18.0))` as a placeholder when no error.

### Printer ComboBox (D-05, D-06)

```rust
egui::ComboBox::from_label("")   // label is placed manually above
    .selected_text(
        state.selected_printer
            .as_deref()
            .unwrap_or("Selecione uma impressora")
    )
    .width(ui.available_width())
    .show_ui(ui, |ui| {
        for printer in &state.printer_list {
            ui.selectable_value(
                &mut state.selected_printer,
                Some(printer.display_name.clone()),
                &printer.display_name,   // e.g. "EPSON TM-T20 (USB)" or "COM3 (Serial)"
            );
        }
    });
```

Item format rule: `"{human name} (USB)"` for Windows spooler printers; `"{COM port} (Serial)"`
for COM ports. The type label is always the trailing parenthetical. The owner never sees
"spooler" or "raw bytes".

Pre-selection (D-06): on window init, if `get_default_printer()` returns Some, set
`state.selected_printer` to the matching display name. If the default printer is not in the
combined list, fall through to the "select" placeholder.

### Empty Printer State (D-07)

When `state.printer_list` is empty:
- ComboBox is replaced by a `Frame::group` info box (same surface color `#262626`).
- Inside: `"Nenhuma impressora encontrada — ligue a impressora e conecte o cabo."` in `Small`,
  muted color `#A3A3A3`.
- Below the box: `"Atualizar lista"` secondary button (default egui dark styling, NOT accent
  fill; 36 pt height; full width).
- "Salvar ativação" / "Ativar" button: disabled (see disabled styling below).

```rust
if state.printer_list.is_empty() {
    Frame::group(ui.style())
        .fill(Color32::from_rgb(38, 38, 38))
        .inner_margin(Margin::same(12.0))
        .show(ui, |ui| {
            ui.label(
                RichText::new(
                    "Nenhuma impressora encontrada — ligue a impressora e conecte o cabo."
                )
                .text_style(TextStyle::Small)
                .color(Color32::from_rgb(163, 163, 163))
            );
        });
    ui.add_space(8.0);
    if ui.add(
        egui::Button::new("Atualizar lista")
            .min_size(vec2(ui.available_width(), 36.0))
    ).clicked() {
        // trigger printer re-enumeration
    }
}
```

### Buttons Row (D-04, D-08, D-09)

Two buttons side by side in a `ui.horizontal` row, full-width split: secondary left (40 %),
primary right (60 %).

**"Imprimir teste" — secondary accent button:**
- Same accent fill `#2563EB` as primary, so it is clearly actionable.
- Disabled when `state.selected_printer.is_none()` OR `state.is_busy`.
- Width: `ui.available_width() * 0.40 - 8.0` (8 pt gap between the two buttons).

**"Ativar" / "Salvar ativação" — primary accent button:**
- Label is `"Ativar"` before first successful validation; changes to `"Salvar ativação"`
  after serial is validated and test-print step is reached (or skipped per D-09).
- Width: `ui.available_width() * 0.60`.
- Disabled when: `state.serial_input.trim().is_empty()` OR `state.selected_printer.is_none()`
  OR `state.is_busy`.

**Accent button styling helper** (apply to both accent buttons when enabled):
```rust
fn accent_button<'a>(label: &'a str) -> egui::Button<'a> {
    egui::Button::new(
        RichText::new(label)
            .text_style(TextStyle::Button)   // Bold 15 pt
            .color(Color32::WHITE)
    )
    .fill(Color32::from_rgb(37, 99, 235))      // normal
    // hover and pressed states are set via ui.visuals_mut() before add()
    // or accepted as egui default lightening/darkening on dark theme
}
```
For hover (`#1D4ED8`) and pressed (`#1E40AF`) states: override
`ui.visuals_mut().widgets.hovered.bg_fill` and `ui.visuals_mut().widgets.active.bg_fill`
immediately before rendering each accent button, then restore defaults. This is the standard
egui pattern for per-widget style overrides.

**Disabled button styling:**
```rust
// When button should be disabled:
ui.add_enabled(false,
    egui::Button::new(
        RichText::new(label)
            .text_style(TextStyle::Button)          // Bold 15 pt
            .color(Color32::from_rgb(97, 97, 97))   // muted label
    )
    .fill(Color32::from_rgb(30, 58, 110))            // accent-disabled fill
    .min_size(vec2(width, 36.0))
);
```

**In-progress / Spinner state (D-04):**
When `state.is_busy == true` (network call in flight or print job dispatched):
- Replace button label with an inline `egui::Spinner` + short text:
  ```rust
  ui.horizontal(|ui| {
      ui.spinner();
      ui.label(
          RichText::new("Aguarde…")
              .text_style(TextStyle::Button)   // Bold 15 pt
              .color(Color32::WHITE)
      );
  });
  ```
- The button itself remains accent-filled and occupies the same width.
- Both buttons disabled during busy state.

### Test-Print Confirmation (D-08, D-09)

After "Imprimir teste" is clicked and the print call returns, show a status line below the
button row (not a modal, not a dialog):

**On print success:**
```rust
ui.add_space(4.0);
ui.horizontal(|ui| {
    ui.label(
        RichText::new("A impressão funcionou? ")
            .text_style(TextStyle::Body)
            .color(Color32::from_rgb(245, 245, 245))
    );
    if ui.button("Sim").clicked() {
        state.test_print_confirmed = Some(true);
    }
    if ui.button("Não").clicked() {
        state.test_print_confirmed = Some(false);
    }
});
```

After `Sim` → show `"Impressora pronta."` in success green `#22C55E`, `Small` style.
The "Salvar ativação" button becomes active.

After `Não` → show `"Tente verificar o papel e o cabo, depois imprima de novo."` in muted
text `#A3A3A3`, `Small` style. "Imprimir teste" button re-enabled. "Salvar ativação" also
enabled (warn-but-allow, D-09): this is NOT a save gate.

**On print hardware failure (D-09):**
```rust
ui.add_space(4.0);
ui.label(
    RichText::new(
        "Não consegui imprimir — confira papel/cabo. Você ainda pode salvar e ativar."
    )
    .text_style(TextStyle::Small)
    .color(Color32::from_rgb(239, 68, 68))
);
```
The "Salvar ativação" / "Ativar" button remains enabled (warn-but-allow path).

### Serial Confirmation — 409 Re-bind (D-02) — DESTRUCTIVE

When Noren returns HTTP 409 ("serial já ativo em outra máquina"), show an inline confirmation
section below the serial error area (not an OS dialog box). This is the only destructive
action in the phase — it invalidates the other machine's token.

```
Este serial já está ativo em outro computador. Migrar a licença para esta máquina?
[Manter atual]   [Confirmar migração]
```

- Heading line: `Body` size, primary text color.
- "Manter atual": default egui button, no accent fill (safe / cancel path — worded to say
  what cancelling preserves).
- "Confirmar migração": **destructive red fill `#EF4444`** with white Bold label — NOT accent
  blue. This visually signals irreversibility and distinguishes it from safe primary actions.
- On confirm: send re-bind request (Noren invalidates old token, mints new).
- This replaces the normal error red line for the 409 case only.

```rust
// Destructive confirm button:
ui.add(
    egui::Button::new(
        RichText::new("Confirmar migração")
            .text_style(TextStyle::Button)   // Bold 15 pt
            .color(Color32::WHITE)
    )
    .fill(Color32::from_rgb(239, 68, 68))     // destructive red, not accent
);
```

All error messages appear below the relevant field in `Small` (13 pt), color `#EF4444`.

## State Machine Summary (for executor reference)

```
Idle
  ├─ serial_input changes → clear serial_error
  ├─ "Atualizar lista" clicked → re-enumerate printers (brief spinner on button)
  └─ "Ativar" clicked (serial non-empty, printer selected, not busy)
       → is_busy = true
       → POST /api/agent/activate
       ├─ 200 OK → agentToken received → state = ValidatedAwaitingTestPrint
       ├─ 404/403 → serial_error = "Serial inválido…"  → back to Idle
       ├─ 409 → show re-bind confirmation block (destructive) → Idle or re-bind flow
       └─ transport error → serial_error = "Sem conexão…" + "Tentar de novo"

ValidatedAwaitingTestPrint
  ├─ "Imprimir teste" clicked → is_busy = true → send test bytes
  │    ├─ ok → show "funcionou?" confirmation → AwaitingTestConfirm
  │    └─ hardware error → show warn message → stay (Salvar still enabled)
  └─ "Salvar ativação" clicked (warn-but-allow) → SaveFlow

AwaitingTestConfirm
  ├─ "Sim" → show "Impressora pronta." → state = ReadyToSave
  ├─ "Não" → show retry hint → state = ValidatedAwaitingTestPrint
  └─ "Salvar ativação" clicked → SaveFlow (allowed from any post-validation state)

SaveFlow (D-15)
  → persist agentToken via DPAPI
  → persist printer + tenant config to SQLite
  → register HKCU Run autostart (warn on failure, D-13)
  → window closes, process exits 0
```

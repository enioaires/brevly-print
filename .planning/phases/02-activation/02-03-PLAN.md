<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DPAPI credential store | agentToken (bearer secret) written to disk; only decryptable by same Windows user (Scope::User). |
| HKCU Run registry | Autostart persistence — user-scoped, no elevation. |
| agent → Noren (HTTPS) | Serial out, agentToken in (re-used from Plan 01 client). |
| 409 re-bind | Destructive: confirming invalidates another machine's token. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Information Disclosure | agentToken at rest | mitigate | Stored ONLY via DPAPI CredentialStore (Scope::User); never written to SQLite; never logged. Serial is NOT persisted (D-10). |
| T-02-09 | Elevation of Privilege | autostart registration | mitigate | `WindowsEnableMode::CurrentUser` (HKCU) — no admin/UAC; user can remove via Task Manager (Pitfall 4). |
| T-02-10 | Spoofing / Repudiation | 409 re-bind invalidates another machine | mitigate | Destructive action gated behind explicit "Confirmar migração" (red, distinct from safe actions); Noren is authoritative and invalidates the old token server-side (D-02). |
| T-02-11 | Denial of Service | winit thread frozen during HTTP | mitigate | Async via rt.spawn + oneshot try_recv; never block_on in the event loop (Pitfall 2) — asserted in acceptance criteria. |
| T-02-12 | Tampering | partial write on exit (token saved, config not) | mitigate | All persistence completes synchronously BEFORE `process::exit(0)` (Pitfall 8). |
| T-02-13 | Information Disclosure | agentToken lingering in process memory | accept | Token held only in window state; process exits 0 immediately after save (short-lived one-shot process). |
</threat_model>

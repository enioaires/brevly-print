<objective>
Close the release/publish loop (D-06 "Distribution Polish"): extend the existing Phase 3 Windows
CI job so one run produces the Velopack UPDATE package (not just the installer), surfaces the exact
`version` / `downloadUrl` / `sha256` triple that Noren's `/api/agent/version` endpoint must serve,

OV cert is an external blocker and is NOT procured here.

plans 01/02; this plan makes PRODUCING a consumable update artifact a single CI run, so once the OV
cert and S3/CF hosting (Noren dependency) land, publishing a signed update is turnkey.

Output: an extended `.github/workflows/ci.yml` Windows job.

| CODESIGN secret → signtool | Signing material crosses only when the secret is present; absent on PRs/pre-cert |

| T-7-CI-sign | Spoofing | unsigned installer distribution | transfer | Authenticode signing stays gated on the OV cert secret (Phase 3 D-12); procurement is an external blocker tracked in STATE, not this plan |

| T-{phase}-SC | Tampering | tool installs (vpk via dotnet tool) | accept | `vpk` install is the pre-existing Phase 3 step (`dotnet tool install -g vpk --version "1.*"`), unchanged; no new package added |
</threat_model>

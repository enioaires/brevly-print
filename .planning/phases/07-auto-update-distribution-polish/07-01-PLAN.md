## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Noren backend ‚Üí agent (`GET /api/agent/version`) | Remote version manifest crosses into the agent ‚Äî untrusted until validated |
| S3/CDN ‚Üí agent (`downloadUrl` artifact) | Downloaded binary bytes are untrusted until SHA256-verified |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-V6 | Tampering | Downloaded artifact bytes (`try_check_and_stage`) | mitigate | `verify_sha256(bytes, expected_hex)` (V6 cryptography, RustCrypto `sha2`) computed BEFORE any staging; mismatch ‚Üí `Ok(false)` abort (Task 1 + Task 2 ordering; Task 3 SC-2 test) |
| T-7-V5 | DoS | `check_for_update` semver parse, `verify_sha256` hex length | mitigate | `semver::Version::parse` returns `Err` (never panics) on malformed version; `verify_sha256` rejects wrong-length/malformed hex ‚Äî both unit-tested (Task 1) |
| T-7-mismatch | Tampering / Elevation | SC-2 abort path | mitigate | On SHA256 mismatch, no stage, no `UpdateStaged`, running agent untouched ‚Äî asserted by the Task 3 integration test |
| T-7-DoS | DoS | update task on HTTP failure | mitigate | `try_check_and_stage` returns `Err` (logged + retried next tick), never panics ‚Äî Task 3 no-panic test |
| T-7-V2 | Info Disclosure | `agent_token` in `check_version` | mitigate | Token passed ONLY via `.bearer_auth()`; never in `eprintln!`/`format!`/`bail!` (T-02-02) ‚Äî Task 2 acceptance criterion greps for leaks |
| T-{phase}-SC | Tampering | npm/pip/cargo installs | accept | No new crates installed ‚Äî `sha2 = "0.11.0"` and `semver = "1.0.28"` are already explicit in Cargo.toml and audited [OK] in RESEARCH.md ¬ß Package Legitimacy Audit |

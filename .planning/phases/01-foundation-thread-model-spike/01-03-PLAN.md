<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user (keyboard/window) → agent | The spike window accepts local keyboard input into an egui text field. Input is echoed into a label only — not persisted, not sent anywhere, not used in any query. No injection surface. |
| GPU driver / WARP → agent | wgpu talks to the DX12 driver (or WARP software rasterizer). Local, trusted; failure degrades to a render error, handled by not presenting rather than crashing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-04 | Denial of Service | wgpu adapter request fails (no GPU / no WARP) | mitigate | In CI, `WGPU_BACKEND=dx12` forces WARP (Pitfall 3); interactive proof is on the Windows box with a real display. Adapter failure surfaces as a handled error at startup, not a silent hang. |
| T-1-01 | Denial of Service | credential load at startup after DPAPI key loss (M7) | mitigate | main() matches `CredentialError::{NotFound,Corrupt}` and logs "needs activation" instead of panicking — the exact Phase 2 re-activation hook. Reuses the plan-02 mitigation; verified here by the startup log branch. |

## Not applicable
The spike text field is not an authentication or network input (Phase 2/4). No secrets are rendered. tray-icon (an additional Win32 surface) is deliberately out of scope until Phase 3.
</threat_model>

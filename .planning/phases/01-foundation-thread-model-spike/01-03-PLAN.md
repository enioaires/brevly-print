## Phase Goal

**As a** Brevly Print developer, **I want to** run one binary that opens an egui window on a
winit event loop and, at startup, exercises the real SQLite + credential stores end-to-end,
**so that** the highest-risk unknown (the render/thread model) is proven on a validated,
cross-platform base before any activation UI is built on top of it.

<objective>
Deliver the walking skeleton (D-07 revised): the thinnest end-to-end slice that proves the
whole stack. `src/main.rs` owns a `winit 0.30` `ApplicationHandler` event loop that (a) runs a
velopack stub first, (b) initializes the app dir, migrates `state.db` and writes+reads one
`config` row, and does one credential round-trip through the `CredentialStore` trait, then
(c) opens a raw-`egui` window (via `egui-winit` + `egui-wgpu`) showing one interactive frame:
a text field + a button that updates a visible label. Runs on Linux first (Vulkan/GL), then
is confirmed on Windows (DX12). This closes SC-1.

Purpose: SC-1 is the phase's reason to exist — prove `winit` + raw `egui` integrate without a
second Win32 loop (pitfall C2), on a cross-platform base. The window becomes the real `src/`
foundation Phase 2 grows into the activation screen (D-05), not throwaway.

Output: A `cargo run` that boots the full startup path and opens an interactive egui window,
plus a headless-safe wgpu smoke test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local disk ↔ app | Startup touches `state.db` + `credential.bin` in the app dir; no network, no external input in Phase 1 |
| display/GPU ↔ app | winit window + wgpu surface (Vulkan/GL on Linux, DX12 on Windows) — local, no remote surface |

**Note:** This plan adds no new external attack surface. The startup credential round-trip uses
a hardcoded dummy value (`b"skeleton-dummy"`); real tokens arrive in Phase 2 activation. The
window takes local keyboard input only (no untrusted parsing).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | DoS | startup credential round-trip | mitigate | Uses the typed `CredentialError` path from 01-02 — a missing/corrupt credential logs a typed error and does not crash the skeleton (M7 hook). Dummy value only in Phase 1. |
| T-1-04 | Tampering | velopack update bootstrapper stub | accept | Phase 1 only calls `VelopackApp::build().run()` as a no-op bootstrapper entry (Windows-gated); real update integrity (SHA256) is Phase 7. No update is fetched in Phase 1. |
</threat_model>

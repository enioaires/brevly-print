## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem â†’ agent | `state.db` and `credential.bin` are read/written under `%APPDATA%\BrevlyPrint\` (current user only). |
| Windows DPAPI (CNG) â†’ agent | `credential.bin` ciphertext is bound to the Windows user SID via `CryptProtectData` (Scope::User). The plaintext agentToken exists in memory only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Denial of Service | `CredentialStore::load()` after DPAPI key loss (Windows reinstall / new SID â€” M7) | mitigate | `decrypt_data` failure maps to typed `CredentialError::Corrupt` (never panic, D-16); missing file maps to `CredentialError::NotFound`. Phase 2 hooks these to re-enter activation. Proven by `test_corrupt_blob` + `test_missing_file`. |
| T-1-02 | Information Disclosure | agentToken at rest in `credential.bin` | accept/mitigate | Token is only ever written DPAPI-encrypted (Scope::User); other Windows users cannot decrypt it. Plaintext never touches disk. No further control needed for L1. |
| T-1-03 | Tampering | `credential.bin` bytes altered on disk | mitigate | Tampered ciphertext fails `CryptUnprotectData` â†’ `CredentialError::Corrupt`, degrading safely to re-activation rather than loading a forged token. |

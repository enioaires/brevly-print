<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mpsc channel → print worker | Events from the Pusher task (already deduped by Phase 4 fence) |
| print worker → SQLite | Status writes to `printed_jobs` (must precede ack — C4) |
| print worker → printer | Raw ESC/POS bytes forwarded verbatim to hardware |
| print worker → Noren HTTP | Bearer-authed fetch + ack |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Repudiation / Data loss | ack fired before status='printed' write (C4) | mitigate | D-09 order enforced on EVERY path (disabled-type + success); Task 3 statically asserts UPDATE precedes ack. A crash between print and ack leaves row 'pending' → Phase 6 re-delivers → Phase 4 fence dedups. Severity: high — mitigated. |
| T-05-05 | Tampering | ESC/POS bytes → printer (C1 RAW datatype) | mitigate | Worker calls `printer.print_raw()` only; RAW datatype is hardcoded in `WindowsSpoolerPrinter`. No intermediate layer added. Severity: high — mitigated by not touching the print path. |
| T-05-06 | Denial of Service | malformed `enabled_types` JSON crashes worker at startup | mitigate | D-03 fail-safe: `.and_then(...ok()).unwrap_or_default()` → empty = allow-all; never panics. Severity: medium — mitigated. |
| T-05-07 | Information Disclosure | agent_token in worker logs | mitigate | Token only via `.bearer_auth()` inside noren_client fns; worker never formats it. Acceptance criterion asserts absence. Severity: high — mitigated. |
| T-05-08 | Tampering | job_id in SQL string | mitigate | All `conn.execute` use `rusqlite::params![event.job_id]` — parameterised, no interpolation (T-04-07). Severity: high — mitigated. |
| T-05-09 | Elevation / Deadlock | second SQLite connection without WAL → SQLITE_BUSY under concurrent writes | mitigate | D-03: `pragma_update journal_mode=WAL` on the worker's own connection (same as Pusher task). Severity: medium — mitigated. |
</threat_model>

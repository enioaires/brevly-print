<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Binary assets → include_bytes! | RGBA files are embedded at compile time; wrong dimensions would panic at runtime via Icon::from_rgba expect(). |
| Cargo.toml feature flags | Adding two new Win32 feature flags extends the Windows API surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-03-01-01 | Information Disclosure | Tray tooltip text | mitigate | Tooltip hardcoded to status strings only — no tenant ID, no version number, no serial. T3 from context fully addressed in this plan. |
| T-03-01-02 | Denial of Service | RGBA asset truncation / wrong size | mitigate | Icon::from_rgba validates exact byte count for given dimensions; .expect() panics at startup rather than silently showing a corrupted icon. Wc-c check in acceptance criteria enforces 1024 bytes. |
| T-03-01-03 | Tampering | Cargo feature flag expansion | accept | Adding Win32_System_Threading and Win32_UI_WindowsAndMessaging is a minimal, justified expansion. Plan 03-02 uses these for CreateMutexW (single-instance guard) and MessageBoxW (About/Quit dialogs). |
</threat_model>

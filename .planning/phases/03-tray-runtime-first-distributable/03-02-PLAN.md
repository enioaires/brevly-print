Pass `autostart_path` to `AutoLaunchBuilder` instead of `current_exe`. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS → named mutex | CreateMutexW with Local\ prefix is session-scoped; another process running as the same user can open this mutex by name. |
| Tray menu → process exit | Sair confirms before exiting; unintended exit would stop printing. |
| Autostart path → HKCU Run | Registered path must point at the Velopack stub; wrong path = agent doesn't start after reboot. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-03-02-01 | Denial of Service | Named mutex (T1 from context) | mitigate | Mutex name `Local\BrevlyPrintAgent` uses Local\ (session-scoped) — an attacker process running in the same session could hold it. Acceptable for a per-user tray app; the mutex name is not a secret but also not guessable from outside the user session (Local\ is per-session). No mitigatable improvement without elevation. |
| T-03-02-02 | Tampering | HKCU Run autostart entry (T2 from context) | accept | Any process running as the user can overwrite HKCU Run; this is a fundamental Windows design. The agent verifies startup succeeds (credential probe); wrong path → activation mode (not silent failure). Anomaly logging deferred to Phase 6. |
| T-03-02-03 | Information Disclosure | tray_runtime show_about_dialog | mitigate | About dialog shows only version + product name (`env!("CARGO_PKG_VERSION")`); no tenant ID, no serial, no token. T3 addressed. |
| T-03-02-04 | Denial of Service | Unguarded Sair (quit) | mitigate | confirm_quit_dialog() with explicit PT-BR consequence message before event_loop.exit(). D-06 addressed. |
| T-03-02-05 | Tampering | Velopack autostart path mismatch | mitigate | activation_state.rs detects Update.exe sibling → uses grandparent stub. Wrong path would make agent invisible after reboot — task 3 fixes this directly. |
</threat_model>

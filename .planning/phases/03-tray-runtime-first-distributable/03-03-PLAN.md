<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI secrets → PFX cert | OV certificate PFX stored as base64 GitHub secret; decoded to temp file in runner and deleted after signing. |
| vpk → Setup.exe | Velopack packages the release binary; no secrets or credentials embedded. |
| Setup.exe → end user | Authenticode signature verifies publisher identity; SmartScreen reputation builds over time. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-03-03-01 | Information Disclosure | PFX cert in CI environment | mitigate | Cert decoded to `$env:RUNNER_TEMP\cert.pfx` (GitHub-managed runner ephemeral storage); `Remove-Item $pfxPath` runs immediately after signing. Secret never written to workspace or artifact. |
| T-03-03-02 | Tampering | vpk package includes debug symbols or dev secrets | mitigate | `cargo build --release` strips debug info by default (strip = true in Cargo.toml release profile or Rust default). CI must not set any dev env vars that embed into the binary. Accept: no additional runtime checks needed. |
| T-03-03-03 | Spoofing | Unsigned installer before OV cert | accept | D-13: dev installer is explicitly unsigned and labeled dev-only. Windows blocks unsigned binaries in some configurations — users of the dev installer must run it in a dev/test context. Documented in STATE.md. Final SC-4 gated on OV cert procurement. |
| T-03-03-04 | Elevation of Privilege | GitHub Actions permissions | accept | CI job already has `contents: read` or similar; no write-to-release permissions needed here (artifact upload only). PR-triggered runs don't have access to repo secrets (GitHub default) — signtool step will skip on PR CI runs. |
| T4 (context) | Tampering | Unsigned installer SmartScreen block | mitigate | DIST-01 CI pipeline is complete; OV cert procurement tracked in STATE.md. SC-4 verified post-cert. D-14 reputation reality documented in STATE.md for owner awareness. |
| T5 (context) | Information Disclosure | Dev credentials in release binary | mitigate | vpk packs `--release` binary; CI workflow does not set dev env vars. No credential-embedding mechanism exists in the codebase. |
</threat_model>

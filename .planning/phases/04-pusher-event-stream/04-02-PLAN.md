processing, and the tray reflects Connected/Reconnecting. Covers the runtime behavior of
EVT-01 (receive→enqueue), EVT-02 (re-auth per reconnect), EVT-03 (zombie→backoff→reconnect).

Output: `src/pusher/client.rs` with `run_pusher_loop`, `insert_print_job`, and (debug-only)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Pusher WS → agent | untrusted JSON envelopes + app events cross into the parser |
| Pusher task → SQLite printed_jobs | job_id/job_type written; must be parameterized |
| Pusher task → tray (event-loop thread) | must cross only via EventLoopProxy (C2) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | Pusher re-delivery duplicate print | mitigate | C3 fence — `INSERT OR IGNORE INTO printed_jobs`; mpsc send skipped when changes()==0, so a re-delivered job is enqueued at most once (V5) |
| T-04-05 | Spoofing | auth string reuse after reconnect | mitigate | EVT-02 — the connect loop always re-runs pusher_auth with the fresh socket_id; no variable caches the auth string across the outer loop (Pitfall 7) |
| T-04-06 | Denial of Service | zombie WS connection swallowing events | mitigate | D-05 — 30s ping; one missed pong breaks the inner loop and reconnects with backoff (EVT-03); indefinite yellow, never lost (D-07) |
| T-04-07 | Tampering | SQL injection via job_id/job_type | mitigate | `rusqlite::params!` binds values as parameters, never string-concatenated into SQL (V5) |
| T-04-08 | Elevation of Privilege | tray called off the event-loop thread (C2) | mitigate | Pusher task only calls the `send_health` closure → `proxy.send_event`; it never links or calls tray-icon APIs (Pitfall 4) |
| T-04-09 | Information Disclosure | fake-event shim / debug logs in release | mitigate | `#[cfg(debug_assertions)]` compiles the shim out of `--release`; no token or payload logged in the dispatch loop |
</threat_model>
